📘 GitHub Contribution Bot – User Guide

✅ Getting Started

This bot helps automate GitHub commits to maintain your contribution graph and simulate human-like activity.

  • Requires a GitHub Personal Access Token (classic)
  • Use with your own public repositories
  • Commits to any file path you choose (e.g., log.md)

🔑 How to Create a GitHub Personal Access Token (PAT)

To use this bot, you need a GitHub Personal Access Token with the necessary permissions. Follow these steps:

  1. Go to your GitHub settings: Navigate to GitHub Personal Access Tokens.
  2. Click on "Generate new token (classic)".
  3. Give your token a descriptive name (e.g., "GitHub Bot Token").
  4. Set an expiration date (e.g., 90 days or 1 year) or "No expiration" (use with caution).
  5. Under "Select scopes", **check the repo checkbox**. This grants access to your repositories. For read-only heatmaps, you might also need read:user.
  6. Click "Generate token" at the bottom of the page.
  7. **IMPORTANT:** Copy the generated token immediately. You will not be able to see it again! Store it securely.
  8. Paste this token into the "GitHub Personal Access Token (PAT)" field in the bot's "Main Controls" tab.

For more details, refer to GitHub's official documentation on managing personal access tokens.

🛠️ How to Use

  1. Paste your GitHub token (with repo scope)
  2. Select one or more of your repositories
  3. Set the file path and branch (default: main)
  4. Use Manual, Auto Commit, or Safe Mode options

🧠 Safe Mode Explained

Safe Mode simulates real human behavior by:

  • ⏱️ Adding random delays between commits (20–30 minutes)
  • 🔀 Rotating across multiple repos
  • 🧾 Generating natural, varied commit messages
  • 💤 Occasionally skipping commits to avoid bot detection

⚠️ What To Do

  • ✅ Use your own repos
  • ✅ Let commits spread over time
  • ✅ Commit real or test data, not junk
  • ✅ Keep messages unique or random

🚫 What Not To Do

  • ❌ Don’t spam commits every few seconds
  • ❌ Don’t push to other users’ repos
  • ❌ Don’t use stolen tokens or fake accounts
  • ❌ Don’t game contribution stats dishonestly

🧪 Pro Tips

  • 📱 Install the app as a PWA
  • 🧭 Use multi-repo rotation for realism
  • 📊 Use the real heatmap preview to track your growth
  • 🧠 Let Safe Mode handle randomness

💻 Advanced Usage (Dev Mode)

Trigger Safe Mode loop manually:

safeAutoCommitLoop(["CryptoExplor/test-repo", "CryptoExplor/logs"]);

Stop loop manually:

safeModeEnabled = false;

🚫 Can You Get Banned by GitHub for Using This Bot?

🔍 Short Answer: Very Unlikely — If Used Responsibly

Your script does not break GitHub’s Terms of Service as long as:

  • ✅ You’re committing to your own repositories
  • ✅ You’re not automating spam or harmful content
  • ✅ You’re not trying to manipulate public-facing metrics dishonestly (e.g., for applying to jobs)
  • ✅ You’re not doing this at mass scale across multiple accounts

🛑 What Might Trigger a Ban or Flag?

Risk Factor Severity Notes
🔴 Spamming commits to others' repos High Could lead to suspension
🔴 Using fake/stolen tokens High Severe violation
🟠 Creating 100s of fake repos Medium Might trigger anti-abuse detection
🟠 Doing 1000s of commits/hour Medium Rate limits or behavioral flags
🟡 Committing useless content Low Allowed, but might hurt repo credibility
🟡 “Artificial” heatmap filling Low Common, not against rules (if to your own repos)

💡 Best Practices to Stay Safe

Tip Reason
✅ Use your own repos only Full control, no violation
✅ Set reasonable commit intervals (e.g. every 10–60 min) Looks human
✅ Use realistic commit messages Avoids looking bot-generated
✅ Avoid sudden repo bursts (100s of commits in seconds) Could trigger rate limits
✅ Don’t automate stars, forks, watchers These are against GitHub ToS
